# AI Platform Frontend (static)

Это фронтенд (Telegram Mini App + обычный Web), который **может работать без бэкенда** в гостевом режиме.

## Что работает без бэкенда (GUEST)
- **Чат (DEMO)**: локальные ответы, история, автосохранение
- **Вложения**: прикрепление файлов к сообщению (превью картинок/видео, удаление)
- **Экспорт/импорт**: чат, проекты и настройки
- **Проекты**: локальный CRUD + поиск/сортировка
- **Медиа**:
  - **Фото-редактор** (Canvas): rotate + базовые фильтры + paste/drag&drop + отправка в чат
  - **Видео-инструменты**: trim через **FFmpeg.wasm** (подгружается из CDN при первом запуске)
- **Настройки**: theme (light/dark/auto), font size, reduce motion, параметры чата, лимит вложений, API base

## Что требует бэкенд (USER)
- Реальные ответы AI (`/ai/chat` или fallback `/ai/ask`)
- Ключи, напоминания, календарь, уведомления
- Админка

## Важно про «настоящий гостевой режим»
Если ты хочешь, чтобы гость реально мог использовать AI **без регистрации**, это должно делаться в бэкенде:
- гостевой JWT (короткий TTL)
- rate-limit / лимит токенов / лимит вложений
- запрет админ/ключей
- логирование и антиабьюз

## Запуск
### 1) Локально
Открой `index.html` или подними любой статик-сервер.

### 2) Docker + nginx (если нужен прокси на /api/v1)
- `docker-compose up --build`
- nginx проксирует `/api/` на backend, а статику отдаёт сам.

## Настройка API base
В **Настройки → Сеть** можно поставить:
- `/api/v1` (если есть nginx proxy)
- `https://domain/api/v1` (например для GitHub Pages)
